\section{Introduction}\label{sec:intro}

Refinement typing is now a standard way to package \emph{may}/safety reasoning:
an overapproximate qualifier states which outcomes an expression \emph{may} produce.
Its dual, \emph{must}/reachability reasoning, is underapproximate:
the qualifier describes outcomes a computation \emph{must} be able to realize.
Coverage types~\cite{CoverageType} show that this dual reading is useful in practice even when
the goal is not bug finding: one verifies that good behaviors are not merely allowed but
must be attainable.

Recent type-theoretic work also studies incorrectness and refutation directly
through two-sided/refutational typing~\cite{Ill-types,RamsayIncorType}.
Together, these lines establish a clear message:
\emph{must}-style reasoning belongs in type systems, not only in external analysis engines.
Yet, in most refinement-based implementations, core typing and subtyping judgments remain
primarily \emph{may}-oriented; must-style evidence is often delegated to separate search
procedures or post-hoc counterexample tools.

This paper targets that gap at the judgment level.
We seek one higher-order refinement typing discipline that can freely mix over- and
underapproximate modes in function interfaces, while retaining concise, compositional
types for functional programs.
The challenge is not adding one extra modality bit.
The challenge is to control dependencies induced by underapproximate choices across
abstraction and application.

Most of those challenges stem from abstraction and application, not from imagining I/O as a flat global store:
while triple-style accounts classify whole input/output envelopes as uniformly over or under, refinement typing interleaves the two modality families along curried introductions.
For instance, consider a curried refinement type whose argument positions interchange over- and underapproximate qualifiers:
\begin{align*}
    n{:}\ort{\Int}{\top} \sarr \urt{\List[\Int]}{\I{mem}(\vnu,n)} \sarr \urt{\List[\Int]}{\I{hd}(\vnu, n)} 
\end{align*}\noindent
This specification bundles a parameterized coverage obligation: given an arbitrarily tagged integer \(n\), if we supply all possible input lists that contain \(n\), the return type demands that outputs \emph{must} cover every list whose head is \(n\) (one possible implementation is to sort the inputs). This function mixes modality patterns along the spine rather than collapsing either side of the arrow to a uniform over- or underapproximate reading.
Neither curried premise is uniformly ``purely over'' or ``purely under'' in isolation.
Translating arbitrarily through global-store encodings restores at best four coarse corners
(one over/under choice per precondition/postcondition knot) without the fine modality patterns function refinement typing already demands.
Hence the design goal goes beyond attaching one extra modality tag to overapproximation: a unified system must
\emph{compositionally reconcile} arbitrary modality regimes throughout higher-order pipelines,
including coverage typing's hallmark over-parameter with under-return arrangement~\cite{CoverageType}.

Second, refinement typing favors concise interfaces, but underapproximate reasoning is sensitive to hidden dependency.
This tension already appears in canonical incorrectness examples~\cite{OHP19}:
\begin{align*}
    [ \top ] \; \Code{if\; \I{even}(x)\; \&\&\; \I{odd}(y)\; then\; z := 42} \; [\Code{z} = 42 \land \I{even}(\Code{x}) \land \I{odd}(\Code{y})]
\end{align*}\noindent
the postcondition must carry guard information on \(x\) and \(y\), even though only \(z\) is assigned.
By contrast, type interfaces should stay short, for example
\begin{align*}
    x{:}\urt{\Int}{\top} \sarr y{:}\ort{\Int}{\vnu > 0} \sarr \urt{\Int}{\vnu = 42}
\end{align*}\noindent
(treating the return value as the final position) without re-stating every relation among
\(x\), \(y\), and that result.
Whether this abstraction is sound depends on continuation structure:
some uses can treat variables independently, while others rely on joint behavior.
So the type system must track hidden dependency modularly.

This paper builds \emph{strategy types}, a resource-modal refinement type system that keeps
over- and underapproximation as first-class modalities on refinements and function positions.
Operationally, underapproximate hypotheses represent witness choices that are consumed linearly:
once fixed, they cannot be silently renegotiated.
Overapproximate hypotheses remain may-style bounds.
This asymmetry is exactly where dependency control matters in higher-order typing.
At the semantic level, this interpretation aligns with alternating
universal/existential choice (equivalently demonic/angelic choice), but the
typing discipline is formulated directly as a refinement type system.

Our key technical move is \emph{bunched typing}.
Instead of carrying dependency only in qualifiers, we structure contexts with
additive \(\ctcomma\) and separating \(\ctstar\), following bunched typing~\cite{BunchedType}
and resource-sensitive local reasoning principles.
These bunched contexts distinguish assumptions that may share witness state from those that must remain disjoint, enabling a precise ``who may depend on whom'' discipline and framing-style modularity at the type level.

\paragraph{Contributions.}
We summarize the main technical contributions as follows.
\begin{itemize}[leftmargin=*]
\item We present \emph{strategy types}, a resource-modal and bunched refinement type system that freely combines over- and underapproximate modalities at higher-order function positions.
\item We define a strategy-resource semantics (with a corresponding modal strategy logic) that gives a compositional account of approximation, independence, and branching for typed functional programs.
\item From this semantics, we derive declarative typing and subtyping rules that enforce an explicit dependency policy---which assumptions may be correlated, which must be separated, and how those constraints propagate compositionally.
\end{itemize}

\section{Discussion}%
\label{sec:discussion}

Our context type system combines the strengths of refinement type
systems and BI-logic-based verification approaches, enabling
compositional reasoning about semantic resources (in our case, angelic
and demonic capabilities) from expressive, modular specifications of
program behavior. It also inherits many of the challenges of these
systems, including the need for some form of user-provided guidance
and the inherent tension between richness of specification and
decidability.

A syntax-directed typing algorithm, necessary for any efficient
implementation, would require additional typing annotations to
constrain the flexibility of the existing typing rules that can be
otherwise quite unconstrained: the typing rules for match statements
and functions, for example, admit multiple valid typings.  It is
impractical, however, to ask users to write function signatures that
anticipate the requirements imposed by all calling contexts.  Even
putting aside the question of typing annotation burden, type-checking
programs with unrestricted angelic and demonic modalities in the
typing context can lead to formulae with quantifier alternations that
lie outside decidable fragments of first-order logic.

Similar challenges arise in separation-logic-based verification, with
initial proposals requiring substantial user-supplied annotations for
tasks such as frame inference and invariant discovery. Subsequent work
has shown that these costs can be substantially reduced through
automated inference techniques, e.g., bi-abduction~\cite{CD11}, which
enables scalable precondition and frame inference, and underpins
modern automated separation-logic analyzers~\cite{DFLO19}.
Restricting where angelic and demonic modalities can appear can also
help guarantee decidable verification conditions, as has been
demonstrated in related settings~\cite{CoverageType}.  Alternatively,
rather than pursuing full automation, we might consider providing
support for context types within an interactive mechanized
verification system like Rocq in the spirit of frameworks like
Iris~\cite{Iris} or VST~\cite{AD+14}.  In this setting, users would
retain fine-grained control over typing choices, with specialized
context type-aware tactics used to facilitate local type inference,
search, and backtracking.

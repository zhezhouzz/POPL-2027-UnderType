¥section{Formalization}¥label{sec:formal}

¥providecommand{¥upperB}{¥boxed{¥mathsf{o}}}
¥providecommand{¥lowerB}{¥boxed{¥mathsf{u}}}
¥providecommand{¥qtrue}{¥top}

This section presents the semantic foundations of our system in
layers: substitutions and finite dependency structure; the choice
resource algebra; pure propositions embedded in that algebra; choice
logic with approximation modalities; and finally modal refinement
types whose meaning is a choice-logic formula.
Except where noted, definitions follow the most recent formulation in
our working notes (slides of March‾17).

¥subsection{Substitutions and pointwise combination}

¥begin{definition}[Compatible union of partial maps]¥label{def:subst-compat}
  Substitutions are partial maps $¥Code{Var} ¥rightharpoonup ¥Code{Value}$.
  Write $¥mapCompat{¥sigma_1}{¥sigma_2}$ when $¥sigma_1$ and $¥sigma_2$ are
  ¥emph{compatible}, i.e.¥ they agree on the intersection of their domains.
  ¥[
    ¥mapCompat{¥sigma_1}{¥sigma_2}
    ¥doteq
    ¥forall x ¥in ¥Dom{¥sigma_1} ¥cap ¥Dom{¥sigma_2}.¥; ¥sigma_1(x) = ¥sigma_2(x).
  ¥]
  Under $¥mapCompat{¥sigma_1}{¥sigma_2}$, the ¥emph{union} of their graphs is
  again a partial map, written $¥sigma_1 ¥cup ¥sigma_2$ as usual.
 ¥end{definition}

We fix a set $¥Code{Var}$ of variables and a domain $¥Code{Value}$ of
semantic values.
A ¥emph{substitution} is a partial map
$¥subs : ¥Code{Var} ¥rightharpoonup ¥Code{Value}$.
We write $¥Dom{¥subs}$ for its domain.

¥begin{definition}[Resource compatibility and pointwise union]¥label{def:resource-times}
Let $m_1$ and $m_2$ be sets of substitutions.
We reuse $¥mapCompat{¥cdot}{¥cdot}$: for sets, $¥mapCompat{m_1}{m_2}$ means that
every cross-pair of substitutions is compatible (in the sense of
¥autoref{def:subst-compat}):
¥[
  ¥mapCompat{m_1}{m_2}
  ¥;¥stackrel{¥mathrm{def}}{¥Longleftrightarrow}¥;
  ¥forall ¥subs_1 ¥in m_1.¥;¥forall ¥subs_2 ¥in m_2.¥; ¥mapCompat{¥subs_1}{¥subs_2}.
¥]
The ¥emph{pointwise union} $m_1 ¥times m_2$ is the partial binary operation
¥[
  m_1 ¥times m_2 ¥;¥doteq¥;
  ¥begin{cases}
    ¥{¥, ¥subs_1 ¥cup ¥subs_2 ¥mid ¥subs_1 ¥in m_1,¥; ¥subs_2 ¥in m_2 ¥,¥}
      & ¥text{if } ¥mapCompat{m_1}{m_2}, ¥¥
    ¥text{undefined}
      & ¥text{otherwise.}
  ¥end{cases}
¥]
¥end{definition}

Note that for any substitution $¥sigma$, $¥mapCompat{¥sigma}{¥sigma}$ always holds since they always agree on their domain. Similarly, singleton set $¥{¥sigma¥}$ is compatible with itself. Thus, if $m$ is a singleton set, $m ¥times m = m$.

¥begin{definition}[Restriction to a variable set]¥label{def:restriction}
  Fix finite $X ¥subseteq_{¥mathrm{fin}} ¥Code{Var}$.
  For a substitution $¥sigma$, write $¥restrictTo{¥sigma}{X}$ for the usual
  restriction of the partial map‾$¥sigma$ to‾$X$: domain
  $¥Dom{¥sigma} ¥cap X$, and agreement with‾$¥sigma$ on that set (so
  $X ¥subseteq ¥Dom{¥sigma}$ is not required).
  For a set $m$ of substitutions, set
  ¥[
    ¥restrictTo{m}{X} ¥;¥doteq¥; ¥{¥, ¥restrictTo{¥sigma}{X} ¥mid ¥sigma ¥in m ¥,¥}.
  ¥]
¥end{definition}


¥subsection{Choice algebra}

¥begin{definition}[Choice algebra]¥label{def:choice-algebra}
A ¥emph{(flat) choice algebra} is a ¥emph{partial commutative semiring}
$(¥Res,{+},{¥times},¥mathbf{0},¥mathbf{1})$: partial binary operations
${+}$ and‾${¥times}$ and constants $¥mathbf{0},¥mathbf{1}$ such that,
whenever the expressions involved are well-formed, the usual semiring
laws hold---associativity and commutativity of ${+}$ and‾${¥times}$,
identities $¥mathbf{0}$ and‾$¥mathbf{1}$, distributivity of‾${¥times}$
over‾${+}$, and annihilation
$m ¥times ¥mathbf{0} = ¥mathbf{0} ¥times m = ¥mathbf{0}$.
We realize it as follows; elements of‾$¥Res$ are called ¥emph{resources}.
¥begin{itemize}
¥item Each resource $m ¥in ¥Res$ is a ¥emph{set of substitutions}
  (element of $¥mathcal{P}(¥Code{Var} ¥rightharpoonup ¥Code{Value})$) which has the ¥emph{same domain}. We reuse $¥Dom{¥cdot}$ for the domain of a resource, the domain of empty set is empty.
¥item $¥mathbf{0} = ¥emptyset$ and $¥mathbf{1} = ¥{¥emptyset¥}$.
¥item $m_1 + m_2$ is set union, and is well-formed only when
  $¥Dom{m_1} = ¥Dom{m_2}$, where
  $¥Dom{m} ¥doteq ¥bigcup_{¥subs ¥in m} ¥Dom{¥subs}$.
¥item ${¥times}$ is the pointwise union from ¥autoref{def:resource-times},
  well-formed exactly on $¥mapCompat{¥cdot}{¥cdot}$ pairs.
¥end{itemize}
We also equip $¥Res$ with a partial order $¥leq$: for $m_1,m_2 ¥in ¥Res$,
define $m_1 ¥leq m_2$ to hold when there exists a finite set
$X ¥subseteq_{¥mathrm{fin}} ¥Code{Var}$ such that
$¥restrictTo{m_2}{X} = m_1$.
¥end{definition}

¥paragraph{Partial order} The order $¥leq$ is the analogue of ``more information by releasing the variables set constraint''. Note that for any well-defined multiplication $m_1 ¥times m_2$, it always holds that $m_1 ¥leq m_1 ¥times m_2$ and $m_2 ¥leq m_1 ¥times m_2$. It underpins
Kripke-style monotonicity for the multiplicative fragment.

¥begin{example}[Choice algebra]
Let $m_x ¥doteq ¥{¥,[x ¥mapsto 1],¥, [x ¥mapsto 2]¥,¥}$, so $|m_x| = 2$ and
$¥Dom{m_x} = ¥{x¥}$.
Let $m_y ¥doteq ¥{¥,[y ¥mapsto 3],¥, [y ¥mapsto 4]¥,¥}$, with $¥Dom{m_y} = ¥{y¥}$.
Then
¥[
  m_x ¥times m_y
  = ¥left¥{
    ¥begin{array}{@{}l@{}}
      [x ¥mapsto 1,¥, y ¥mapsto 3],¥,
      [x ¥mapsto 1,¥, y ¥mapsto 4],¥¥[¥jot]
      [x ¥mapsto 2,¥, y ¥mapsto 3],¥,
      [x ¥mapsto 2,¥, y ¥mapsto 4]
    ¥end{array}
  ¥right¥}.
¥]
Now take
$m_{xy} ¥doteq ¥{¥, [x ¥mapsto 1,¥, y ¥mapsto 3],¥, [x ¥mapsto 2,¥, y ¥mapsto 4]¥,¥}$.
We have
$¥restrictTo{m_{xy}}{¥{x¥}} = m_x$ and $¥restrictTo{m_{xy}}{¥{y¥}} = m_y$,
so by the definition of‾$¥leq$,
$m_x ¥leq m_{xy}$ and $m_y ¥leq m_{xy}$.
The product satisfies the same restrictions:
$¥restrictTo{(m_x ¥times m_y)}{¥{x¥}} = m_x$ and
$¥restrictTo{(m_x ¥times m_y)}{¥{y¥}} = m_y$.
Nevertheless $|m_x ¥times m_y| = 4 > 2 = |m_{xy}|$; equivalently,
¥[
  m_x ¥times m_y
  = m_{xy} + ¥{¥, [x ¥mapsto 1,¥, y ¥mapsto 4],¥, [x ¥mapsto 2,¥, y ¥mapsto 3] ¥,¥},
¥]
where $+$ is resource addition (set union) from ¥autoref{def:choice-algebra};
the sum is well-formed because $¥Dom{m_{xy}} = ¥{x,y¥}$ matches the domain
union of the second summand.
¥end{example}

¥paragraph{Partial order and addition} The partial order ($¥leq$) is almost completely independent of
addition ($+$) on resources: in particular, when $m_1 + m_2$ is defined we do ¥emph{not}
generally expect the usual monotonicity inequality
$m_1 ¥leq m_1 + m_2$.
This is unlike incorrectness logic‾¥cite{OHP19} and outcome
logic‾¥cite{ZDS23,Zil25TOPLAS}, where branching is tightly coupled to
the consequence relation.
Here, $+$ simply adjoins more substitutions---i.e.¥ it records strictly
more alternatives---and whether that strengthens or weakens a
judgment depends on the approximation mode.
Under an overapproximate interpretation, extra alternatives relax universal
guarantees (more ways a safety property can fail); under an
underapproximate interpretation, they add witnesses for existential or
reachability claims.
Because the two modes are dual and our modalities can mark formulas
with either interpretation, a logic that uniformly enforces Kripke monotonicity
should not identify $¥leq$ with enrichment by‾$+$.
By contrast, expanding or shrinking the set of tracked variables is
orthogonal to over- versus under-approximation: for example, the same
resource
$¥{¥,[x ¥mapsto 0],¥, [x ¥mapsto 1],¥, [x ¥mapsto 2]¥,¥}$
is compatible with an overapproximate interpretation that $x$ may range
over‾$[0,5]$ and, separately, with an underapproximate interpretation that
witnesses for‾$x$ lie in‾$[1,2]$.
Extending every substitution with arbitrary bindings for a fresh
variable‾$y$ disturbs neither kind of judgment.

¥subsection{Basic choice logic}

¥emph{Basic} choice logic adds bunched connectives over resources.
We write $m ¥models p$ for satisfaction of a resource proposition $p$
at world $m ¥in ¥Res$.

¥begin{definition}[Basic choice logic]
Define satisfaction for the non-modal connectives by:
¥begin{itemize}
¥item $m ¥models p * q$ iff $¥exists m_1,m_2.¥; m_1 ¥times m_2 ¥leq m$ and
  $m_1 ¥models p$ and $m_2 ¥models q$.
¥item $m ¥models p ¥wand q$ iff $¥forall m'.¥; m' ¥models p$ implies
  $m' ¥times m ¥models q$.
¥item $m ¥models p ¥oplus q$ iff $¥exists m_1,m_2.¥; m_1 + m_2 ¥leq m$ and
  $m_1 ¥models p$ and $m_2 ¥models q$.
¥item Boolean and first-order structure:
  $m ¥models ¥qtrue$ always;
  $m ¥models ¥qfalse$ iff $m = ¥mathbf{0}$;
  $m ¥models p ¥land q$ iff $m ¥models p$ and $m ¥models q$;
  $m ¥models p ¥lor q$ iff $m ¥models p$ or $m ¥models q$;
  $m ¥models p ¥Rightarrow q$ iff $m ¥models p$ implies $m ¥models q$;
  $m ¥models ¥forall x.¥,p$ iff for all values $v$, $m ¥models p[x¥mapsto v]$;
  $m ¥models ¥exists x.¥,p$ iff for some $v$, $m ¥models p[x¥mapsto v]$.
¥end{itemize}
Pure propositions embed via the coercion above.
¥end{definition}

The connective $¥oplus$ internalizes ¥emph{nondeterministic branching}
as resource sum; $*$ internalizes ¥emph{independent} combination of
partial information.
The magic wand isolates the hypotheses needed for a multiplicative
conclusion, and will appear in the semantics of higher-order
application patterns (argument usage that ``consumes'' part of the
choice budget).

¥subsection{Approximation modalities}

¥begin{definition}[Choice logic with approximation]
Extend the logic with unary modalities $¥upperB$ (overapproximation)
and $¥lowerB$ (underapproximation) by
¥begin{itemize}
¥item $m ¥models ¥upperB ¥phi$ iff $¥exists m_1,m'.¥; m + m_1 = m'$ and
  $m' ¥models ¥phi$;
¥item $m ¥models ¥lowerB ¥phi$ iff $¥exists m_1,m'.¥; m' + m_1 = m$ and
  $m' ¥models ¥phi$,
¥end{itemize}
where $+$ is the partial sum on resources (again, only defined on
equal domains).
¥end{definition}

Thus $¥upperB$ relaxes a constraint by embedding $m$ into a larger
alternative sum, while $¥lowerB$ strengthens it by exhibiting $m$ as a
sum that already includes a witness sub-resource satisfying‾$¥phi$.
The modalities interact cleanly with $¥land$ and $¥oplus$ on pure
approximation-normal forms: for instance, nested uses of $¥upperB$ over
conjunctions collapse to $¥upperB$ applied to the intersection of the
underlying pure propositions, and dually for $¥lowerB$ and unions; we
use those laws tacitly when reasoning about refinement qualifiers.

¥subsection{Approximation normal form}

For logical considerations (and for effective normalization in
implementations), it is convenient to restrict to layered formulas in
which approximation markers apply only to pure propositions.

¥begin{definition}[Approximation normal form (syntax sketch)]
¥[
  p ¥;::=¥; ¥upperB ¥phi ¥mid ¥lowerB ¥phi ¥mid
      p ¥land p ¥mid p ¥lor p ¥mid p ¥Rightarrow p ¥mid
      ¥qtrue ¥mid ¥qfalse ¥mid
      p * p ¥mid p ¥oplus p ¥mid p ¥wand p ¥mid
      ¥forall x.¥, p ¥mid ¥exists x.¥, p
¥]
with $¥phi$ ranging over pure propositions
(¥autoref{subsec:pure-prop}).
¥end{definition}

¥noindent
(We omit a full normalization theorem here; the slides record the
equivalences that justify pushing $¥upperB/¥lowerB$ through
propositional connectives.)

¥paragraph{Dependent products (informal).}
When refinements mention earlier variables in a separating context
(e.g.¥ ``choose $y$ in a range that depends on the witness for $x$''),
plain $¥times$ on flat resources only describes rectangular
dependence.  The general fix, as in our March‾17 notes, introduces
¥emph{monadic} resources $m : (¥Code{Var} ¥rightharpoonup ¥Code{Value})
¥to ¥mathcal{P}(¥Code{Var} ¥rightharpoonup ¥Code{Value})$ whose output
substitutions may depend on an ¥emph{input} substitution, with $¥times$
computed fiberwise while preserving read-only parts of the input.
We keep working with the flat carrier in this paper fragment, and
defer the monadic layering to the technical development of dependent
separating contexts in types.

¥subsection{Pure propositions}¥label{subsec:pure-prop}

Before adding approximation, we assume a class of ¥emph{pure}
propositions whose semantics are ¥emph{sets of substitutions}.

¥begin{definition}[Pure propositions --- syntax fragment]
Let atomic tests range over standard value predicates.
A minimal grammar adequate for examples below is
¥[
  ¥phi ¥;::=¥; ¥mathsf{atom} ¥mid ¥Dom{X} ¥mid ¥phi ¥cap ¥phi
    ¥mid ¥phi ¥cup ¥phi ¥mid ¥phi ¥setminus ¥phi ¥mid ¥phi ¥times ¥phi
¥]
where $¥Dom{X}$ asserts that every substitution in the denotation
has domain exactly‾$X$.
Each $¥phi$ denotes a set $¥denot{¥phi}$ of substitutions.
¥end{definition}

¥begin{definition}[Embedding into resources]
Given $¥denot{¥phi}$, we reuse the name $¥phi$ for the associated
resource proposition, and write $m ¥models ¥phi$ when
$¥denot{¥phi} ¥leq m$ in the sense of the order above (with the
obvious coercion of a set of substitutions into a resource).
¥end{definition}

¥subsection{Modal choice types: syntax}¥label{sec:choice-types-syntax}

We now introduce the type syntax interpreted by translation into
choice logic.
Let $b$ range over base types (e.g.¥ $¥Code{unit}$, $¥Code{bool}$,
$¥Code{int}$).

¥begin{definition}[Modal refinement types]
¥[
¥begin{array}{ll}
  ¥phi ¥;::= & ¥text{literal} ¥mid ¥qtrue ¥mid ¥qfalse ¥mid ¥neg ¥phi
      ¥mid ¥phi ¥land ¥phi ¥mid ¥phi ¥lor ¥phi
      ¥mid ¥phi ¥Rightarrow ¥phi
      ¥mid ¥phi ¥times ¥phi ¥mid ¥forall x.¥,¥phi ¥mid ¥exists x.¥,¥phi
  ¥¥
  ¥tau ¥;::= &
      ¥ort{b}{¥phi} ¥mid ¥urt{b}{¥phi} ¥mid ¥tau ¥oplus ¥tau
      ¥mid x{:}¥tau ¥sarr ¥tau ¥mid x{:}¥tau ¥wand ¥tau ¥mid x{:}b ¥garr ¥tau
  ¥¥
  ¥Gamma ¥;::= &
      ¥emptyset ¥mid x{:}¥tau ¥mid ¥Gamma ¥sep ¥Gamma ¥mid ¥Gamma,¥Gamma
¥end{array}
¥]
Here $¥ort{b}{¥phi}$ is an overapproximate refinement and
$¥urt{b}{¥phi}$ underapproximate; $¥oplus$ on types mirrors resource
sum at the kind level; $¥sarr$ and $¥wand$ distinguish
multiplicative versus ``frame-preserving'' dependency in function
abstraction (as in the semantic clauses below); and $x{:}b ¥garr ¥tau$
is a dependent telescope familiar from refinement-type presentations.
The context former $¥Gamma_1 ¥sep ¥Gamma_2$ uses the separating
combinator (syntax for $*$ at the context level), while ``,'' is the
ordinary additive conjunction of hypotheses.
¥end{definition}

¥subsection{Denotational semantics}

We assume a fixed operational semantics for programs, writing
$¥mstep{e}{v}$ when closed $e$ reduces to value $v$.
Let $¥vnu$ be a distinguished name for the value placeholder in
qualifiers.

¥begin{definition}[Qualifier and term interpretations]
For qualifier $¥phi$, define
$¥denot{¥phi}$ as the set of substitutions $¥subs$ such that
$¥subs ¥models ¥phi$ under the standard interpretation of first-order
tests.
For closed $e$, define the singleton-union resource for value equality
¥[
  ¥denot{¥vnu = e} ¥;¥doteq¥; ¥{¥, [¥vnu ¥mapsto v] ¥mid ¥mstep{e}{v} ¥,¥}.
¥]
¥end{definition}

¥begin{definition}[Type and context interpretations]
For type $¥tau$, the object $¥denot{e : ¥tau}$ is a resource
proposition (possibly with embedded first-order quantifiers over
argument expressions).
The key clauses are:
¥begin{itemize}
¥item $¥denot{e : ¥ort{b}{¥phi}} ¥;¥doteq¥;
      ¥denot{¥vnu = e} ¥Rightarrow ¥upperB ¥denot{¥phi}$.
¥item $¥denot{e : ¥urt{b}{¥phi}} ¥;¥doteq¥;
      ¥denot{¥vnu = e} ¥Rightarrow ¥lowerB ¥denot{¥phi}$.
¥item $¥denot{e : ¥tau_1 ¥oplus ¥tau_2} ¥;¥doteq¥;
      ¥denot{e : ¥tau_1} ¥oplus ¥denot{e : ¥tau_2}$.
¥item For functions, with $e_x$ ranging over closed instantiations of
  the parameter,
  ¥[
    ¥denot{e : x{:}¥tau_x ¥sarr ¥tau}
      ¥;¥doteq¥;
      ¥forall e_x.¥;
      ¥denot{e_x : ¥tau_x} ¥Rightarrow
      ¥bigl(¥denot{e_x : ¥tau_x} ¥land
            ¥denot{e¥,e_x : ¥textstyle¥bigoplus_{¥mstep{e_x}{v_x}} ¥tau[x ¥mapsto v_x]}¥bigr),
  ¥]
  where the sum ranges over values reachable from $e_x$, and $¥tau[x
    ¥mapsto v_x]$ applies the semantic substitution to the return
    refinement.
  Dually,
  ¥[
    ¥denot{e : x{:}¥tau_x ¥wand ¥tau}
      ¥;¥doteq¥;
      ¥forall e_x.¥;
      ¥denot{e_x : ¥tau_x} ¥wand
      ¥denot{e¥,e_x : ¥textstyle¥bigoplus_{¥mstep{e_x}{v_x}} ¥tau[x ¥mapsto v_x]}.
  ¥]
¥end{itemize}
Contexts are interpreted as functions producing propositional
obligations over an incoming substitution: $¥denot{¥emptyset}$ is the
constant $¥qtrue$; $¥denot{x{:}¥tau}$ threads the substitution into
$¥denot{x : ¥tau}$; additive and separating context combination mirrors
$¥land$ and $*$ at the meta-level (cf.¥ the March‾17 slides for the
exact $¥lambda$-lifting).
¥end{definition}

¥begin{definition}[Typing judgment (semantic)]
For closed $e$, say ¥emph{$e$ has type $¥tau$ under context $¥Gamma$}
whenever
¥[
  ¥denot{¥Gamma}(¥emptyset) ¥;¥models¥; ¥denot{e : ¥tau}.
¥]
¥end{definition}

The function clauses make the distinction between ``argument coverage
that can be reused when checking the application'' ($¥sarr$) and
``coverage treated as a hypothesis that must be framed away before the
postcondition is checked'' ($¥wand$) internal to choice logic itself.
This is the point where algebraic resource structure meets
higher-order programs: neither incorrectness triples nor outcome
predicates over stores alone expose the $¥wand$/$¥sarr$ distinction at
the granularity of function introductions.
